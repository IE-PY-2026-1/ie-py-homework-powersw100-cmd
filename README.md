# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
개인 맞춤형 피트니스 및 식단 플래너
<br>


## 2. 시나리오 (5~10줄)
이 프로그램은 사용자의 건강한 체중 관리와 체계적인 식단을 도와주는 '퍼스널 AI 코치' 시스템입니다. 가장 먼저 사용자의 이름, 나이, 키, 현재 체중과 같은 신체 지표를 입력받아 현재의 BMI(체질량 지수)를 계산하고, 목표 체중까지 얼마나 더 감량(또는 증량)해야 하는지 분석하며 관리를 시작합니다. 이후에는 사용자가 하루 동안 먹은 음식들을 리스트에 차곡차곡 담아 저장하고, 현재 BMI 상태에 따라 오늘 섭취한 칼로리가 적절한지 혹은 과한지를 날카롭게 판정해 줄 것입니다. 나중에는 단순히 데이터를 입력하는 것에 그치지 않고, 사용자가 메뉴판에서 원하는 기능(정보 수정, 식단 기록, 전체 통계 보기 등)을 자유롭게 선택할 수 있는 시스템을 구축할 예정입니다. 최종적으로는 모든 데이터를 딕셔너리 구조로 고도화하여 음식별 칼로리를 정확히 매칭하고, 함수 모듈화를 통해 누구나 쉽게 자신의 건강 상태를 한눈에 파악할 수 있는 완성도 높은 건강 관리 소프트웨어를 목표로 합니다
<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
(예시)
1.사용자 신체 정보 등록 및 수정
2.BMI 지수 및 목표 체중 분석
3.오늘의 식단 기록 추가 (리스트 저장)
4.현재 상태별 식단 적정성 판정
5.전체 건강 관리 기록 조회
6.프로그램 종료

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
이번 과제는 사용자의 신체 데이터를 바탕으로 비만도를 판정하고 식단을 기록하는 '건강 정밀 진단 시스템' 구현입니다. 사용자로부터 이름, 키, 몸무게, 목표 체중을 입력받아 리스트에 저장하고, BMI 지수를 활용하여 현재의 건강 등급을 도출하는 로직을 구축하는 것입니다. 특히 if-elif-else 조건문과 논리 연산자를 활용하여 상황에 맞는 맞춤형 건강 조언을 부여하는 지능형 시스템을 완성해야 합니다.input()으로 사용자의 이름, 나이, 키, 몸무게, 목표 체중을 입력받아 리스트에 저장.BMI 지수 산출 공식을 적용하여 수치 계산: 체중(kg) / (키(m) \times 키(m))if-elif-else를 사용하여 저체중, 정상, 과체중, 비만 등 4단계 건강 등급 판정 로직 추가.and 연산자를 활용하여 '목표 달성 임박' 등 특정 조건에 맞는 특별 응원 메시지 출력 기능 구현..
    
### **🤖 AI 파트너십 과정**
내용 1: 피트니스 매니저 컨셉 및 로직 도출
프롬프트 요약: "건강 관리 프로그램을 만들 거야. 1학년 수준에서 리스트와 조건문을 활용해 BMI 등급을 매기고 식단을 관리하는 재미있는 로직을 추천해 줘."
적용 내용: AI가 제안한 'BMI 기반 맞춤형 건강 등급 판정' 방식과 '목표 체중과 현재 체중의 차이에 따른 동기부여 메시지'라는 아이디어를 실제 코드에 반영함.
내용 2: 효율적인 리스트 활용법 자문
프롬프트 요약: "파이썬 기초 문법 단계(리스트 → 조건문 → 반복문 → 함수/딕셔너리)에 맞춰, '피트니스 플래너' 컨셉의 프로그램이 어떻게 발전할 수 있을지 4단계 기능 확장 로드맵을 제안해 줘."
적용 내용: 단순 데이터 입력을 넘어, **'기초 정보 수집(1차) → 건강 등급 및 식단 판독기 구현(2차) → 무한 메뉴 시스템 구축(3차) → 딕셔너리를 활용한 음식 칼로리 관리(4차)'**로 이어지는 논리적인 확장 로드맵을 확립함
    
### **🛠️ Troubleshooting & 기술 회고:**
문제 1: 키와 몸무게 계산 시 발생하는 TypeError
원인: input()은 모든 입력을 문자열(str)로 받기 때문에 거듭제곱 연산(**2)이나 나눗셈 같은 산술 연산이 불가능했음.
해결: float() 함수를 사용하여 입력값을 실수형으로 형변환하여 계산 로직을 정상화함.
문제 2: 조건문 범위 설정 및 문법 오류
원인: if bmi < 18.5와 같은 조건식에서 부등호 방향을 잘못 설정하거나 끝에 콜론(:)을 빠뜨려 SyntaxError가 발생함.
해결: 파이썬 특유의 들여쓰기(Indentation) 규칙과 모든 조건문 끝에는 항상 :이 들어가야 한다는 규칙을 다시 확인하여 수정함.
